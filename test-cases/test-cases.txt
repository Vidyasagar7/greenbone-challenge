SECTION 3 – PRIORITISED TEST CASES
─────────────────────────────────────────

Priority 1 – TC-01: Login with valid credentials
  Reason: Authentication is the gateway to the entire application. Every other test case
  depends on a successful login. A broken login means no user can access any feature —
  immediate P0 impact across all user journeys.

Priority 2 – TC-02: Complete end-to-end purchase flow
  Reason: This is the core business-critical path. It covers add-to-cart → checkout →
  confirmation in a single flow. If any step is broken, revenue generation stops entirely.
  Highest business risk if not caught.

Priority 3 – TC-03: Add product to cart updates cart badge count
  Reason: Adding to cart is the most common user action and a prerequisite for checkout.
  A bug here (silent failure, wrong count) directly affects conversion. Isolated from the
  full checkout flow so failures are easy to pinpoint.

Priority 4 – TC-06: Remove item from cart
  Reason: Cart management is a critical user action — users routinely change their minds.
  A broken remove function leads to frustration and cart abandonment. Paired naturally
  with add-to-cart.

Priority 5 – TC-07: Checkout form validation (empty required fields)
  Reason: Form validation prevents bad data from entering the checkout flow. An absent
  validation could allow incomplete orders through, causing downstream processing errors.
  Important for data integrity and UX.

Priority 6 – TC-04: locked_out_user sees error message on login
  Reason: Security and access control — the app must correctly block restricted users
  and communicate why. A missing or misleading error could confuse support teams or
  indicate a broken access-control mechanism.

Priority 7 – TC-05: Sort products by price (low to high)
  Reason: Sorting is a usability feature that improves discoverability and conversion.
  A broken sort order leads to user confusion but doesn't block purchases. Medium risk.

Priority 8 – TC-08: Product detail page shows correct information
  Reason: Incorrect product data (wrong price, description mismatch) could mislead
  buyers, but the user can still complete a purchase. Important for data integrity but
  lower blast radius than core flow failures.

Priority 9 – TC-09: Logout clears session and returns to login page
  Reason: Session management is important for security (especially on shared devices)
  but failure here is less immediately disruptive to active users. Still important for
  compliance and security posture.

Priority 10 – TC-10: Cart persists items when navigating away and back
  Reason: State persistence is expected behaviour but edge-case relative to the happy
  path. A bug here is noticeable but unlikely to completely block a purchase. Lowest
  risk among the identified cases.
